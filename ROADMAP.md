# 개발자 웹 이력서 제작 로드맵

## 기술 스펙
- HTML5
- CSS3
- JavaScript (Vanilla JS)
- Tailwind CSS

## 프로젝트 목표
반응형 1페이지 웹 이력서를 제작한다. (모바일 / 태블릿 / 데스크탑 대응)

---

## 1단계 - 프로젝트 설정
- [ ] 프로젝트 폴더 구조 생성 (`index.html`, `css/`, `js/`, `assets/`)
- [ ] Tailwind CSS 설치 및 빌드 환경 구성 (CDN 또는 CLI 방식 중 선택)
- [ ] 기본 HTML 뼈대 작성 (`<head>` 메타 태그, 폰트, 파비콘 등)
- [ ] Git 저장소 초기화 (선택)

## 2단계 - 이력서 콘텐츠 정의
간단하고 일반적인 구성으로 아래 섹션을 준비한다.

- [ ] **헤더(Header)**: 이름, 직무(예: Frontend Developer), 프로필 사진, 한 줄 소개
- [ ] **연락처(Contact)**: 이메일, 전화번호, GitHub, 블로그/포트폴리오 링크
- [ ] **자기소개(About Me)**: 3~4문장 정도의 간단한 소개글
- [ ] **기술 스택(Skills)**: 사용 가능한 언어/프레임워크/툴 목록 (뱃지 or 태그 형태)
- [ ] **경력/프로젝트(Experience & Projects)**: 회사명 또는 프로젝트명, 기간, 담당 역할, 주요 성과 (2~3개)
- [ ] **학력(Education)**: 학교명, 전공, 졸업(예정)년도
- [ ] **자격증/수상(Certificates, 선택)**: 있을 경우 간단히 나열
- [ ] **푸터(Footer)**: 저작권 표시, SNS 링크 아이콘

## 3단계 - 마크업(HTML) 작업
- [ ] 시맨틱 태그로 섹션 구성 (`header`, `nav`, `section`, `footer` 등)
- [ ] 접근성 고려 (alt 속성, aria-label 등)
- [ ] 각 섹션에 앵커 링크 연결 (스무스 스크롤용)

## 4단계 - 스타일링(Tailwind CSS)
- [ ] 색상/폰트 등 기본 테마 정의 (tailwind.config.js 커스터마이징)
- [ ] 반응형 그리드/플렉스 레이아웃 적용 (`sm:`, `md:`, `lg:` 브레이크포인트)
- [ ] 다크모드 지원 (`dark:` 클래스, 선택 사항)
- [ ] 호버/포커스 인터랙션 스타일 (`hover:`, `focus:`)

## 5단계 - 인터랙션(JavaScript)
- [ ] 네비게이션 스무스 스크롤 구현
- [ ] 다크모드 토글 버튼 기능 구현 (선택)
- [ ] 스크롤 시 헤더 고정/하이라이트 효과 (선택)
- [ ] 모바일 햄버거 메뉴 토글 기능
- [ ] 스킬/프로젝트 섹션 등장 애니메이션 (Intersection Observer, 선택)

## 6단계 - 최적화 및 배포
- [ ] 이미지 최적화 (WebP 변환, 적절한 사이즈)
- [ ] Tailwind 빌드 시 미사용 CSS 제거(purge) 확인
- [ ] 크로스 브라우저 테스트 (Chrome, Safari, Edge)
- [ ] 반응형 테스트 (모바일/태블릿/데스크탑)
- [ ] GitHub Pages, Vercel, Netlify 등으로 배포

## 7단계 - 마무리
- [ ] README.md 작성 (프로젝트 소개, 사용 기술, 배포 링크)
- [ ] 코드 정리 및 주석 최소화
- [ ] 최종 점검 후 이력서 링크 공유
